




















































































Eclipse STS is software used to develop web applications using Spring Boot and Java.

Bugzilla is software used to track bugs (software errors)



Installation for Spring Boot STS (Spring Tool Suite)

Method 1: If Eclipse present then just add STS plugin
Method 2: Download STS from Chrome then open directly
 
Method 1 :-

Step 1 — Open Eclipse
Step 2 — Go to Marketplace -> Click Help then Click Eclipse Marketplace
Step 3 — Search Spring Tools by typing in searchbar
Step 4 — Install STS, find spring tools and click install then restart
After restart, Eclipse becomes a Spring Tool enabled IDE

Method 2 :— 
Step 1 — Open chrome and search for 'Spring Tool Suite download'
Step 2 — Download STS by going to the spring tools website and clicking download for compatible version
         The STS alone is a full IDE package 
Step 3 — Extract the zip file
Step 4 — Open STS by clicking on STS.exe or eclipse.exe and when STS opens, installation process is done


Creating Spring Boot project on Eclipse

Step 1: Open Eclipse STS and launch Spring Tool Suite, then select workspace and click launch
Step 2: Create Spring Boot Project by click file, then new, then Spring Starter Project

Here enter this information:
Project Name: demmo
Type: Maven
Java Version: 17
Click next then select Spring Web and click finish

Step 3: Create the Controller

In Package Explorer:
src/main/java -> com.example.demo
Right-click package -> New -> Class
Class Name: HomeController
and click finish

Step 4: Write Code

package com.example.demo;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;
@RestController
public class HomeController {
    @GetMapping("/")
    public String home() {
        return "Application Running Successfully";
    }
}

Step 5: Run Application as Spring Boot App
Step 6: Verify Output by going to http://localhost:8080
Output:
Application Running Successfully
Application developed successfully using Eclipse STS


BUGZILLA (BUG TRACKING)

Bugzilla is not inside STS as it is a web-based tool

Step 7: Open Bugzilla - Open browser then go to Bugzilla site (https://bugzilla.mozilla.org)

Step 8: Login/Create Account to login to Bugzilla dashboard

Step 9: Create New Bug - Click file a bug then select product (web application)

Enter these details:

Summary: Home page not loading properly
Description: Application home page is not displaying expected output
Steps to Reproduce: Open browser, Enter localhost:8080, Page shows incorrect output
Expected Result: Page should show correct message
Actual Result: Incorrect output shown
Select Severity

Then click on submit bug

Step 10: Bug ID Generated

Bugzilla generates Bug ID:
Bug #101
and message 'Bug successfully recorded'


















Scenario: Online Banking Application

Developer builds application using Eclipse STS
Tester tests application
Tester finds issue
Tester logs bug using Bugzilla
Developer checks Bugzilla
Developer fixes issue in STS
Tester retests and closes bug

Example:
Tester finds login issue in Firefox
Tester reports bug in Bugzilla
Manager assigns bug to developer
Developer fixes bug
Status changes to FIXED → CLOSED

OUTPUT 
Spring Boot application executed successfully
Application accessed via browser
Bug created and tracked using Bugzilla

 RESULT
Thus, Eclipse STS was used to develop a Spring Boot application and Bugzilla was used to track software bugs successfully.
